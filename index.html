FuncyTag & FuncyStyle

Generate HTML/JS/CSS from nested Javascript functions--that's all.
FuncyTag & FuncyStyle allow all components of a web page (HTML, CSS, and Javascript) to be generated from purely functional source that still looks pretty much like standard HTML/CSS/JS. Easy to read, easy to write, easy to bundle, easy to reuse.

What is this?

This is an experiment to answer the questions: What if web pages could be developed purely in JavaScript? Under what circumstances does that make development easier?

What do FuncyTag & FuncyStyle do?

FuncyTag & FuncyStyle generate HTML and CSS and Javascript event handlers and everything else a browser for presentation, directly from nested, functional javascript.

Why? What's wrong with traditional HTML+CSS+JS?

HTML+CSS+JS are great and are the go-to solution for most circumstances. But there are times when thinking in three different languages, and juggling three different source files gets really annoying. At those times, it can be really convenient to combine all of those components into a single Javascript function call that acts like a tag with it's own styling and event behaviors.

So what? Should I be impressed?

Not really. This is pretty simple stuff, and that's kind of the point: making stuff as simple and boring for the javascript writer as possible.

There's nothing super novel about noticing the similarities between closing tags and the ending parenthesis on a function call, or similarities between html tag attributes and properties of a javascript object. That's been done a lot of times. Although I think it's kind of keen to represent style="font-size:14px" as cssFontSize_px=14 because doing so has saved me a lot of tedious string concatenation code.

But let's take it further. Can we create more complex web code, and still continue to remain in pure javascript, and continue to make FuncyTag look boring and unimpressive?
